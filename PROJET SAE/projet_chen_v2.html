Dans ce document, nous nous intéressons à l’étude statistique de la recherche de points influents et atypiques de l’IMG. En identifiant ces points, nous pouvons comprendre les patterns et les comportements inhabituels des observations. Et ainsi, éviter le biais des résultats de l’étude.

Introduction : Indice de Masse Grasse

L’Indice de Masse Grasse (IMG) est un indice, exprimé en pourcentage, permettant de juger de la proportion de tissus adipeux d’une personne adulte, qui rend compte de la disproportion entre la masse de graisse et celles des muscles. De nombreuses méthodes existent permettantd’estimer la proportion de masse adipeuse dans le corps humain. Nous allons nous intéresser plus particulièrement à deux façons de calculer cet indice chez l’adulte :

    1. La formule proposée par Brozek est : \(IMG = \frac{457}{densité_corporelle} − 414.2\)
    1. La formule proposée par Siri est : \(IMG = \frac{495}{densité_corporelle} − 450\)

On cherchera donc à expliquer les deux variables :

  1. body.fat : obtenu à l’aide de la fomule de Brozek.
  2. body.fat.siri : obtenu à l’aide de la fomule de Siri.

Détections des points influents ou atypiques

Dans un premier temps, on souhaite identifier les observations qui peuvent fausser les résultats de la régression. Cela consiste à détecter les points influents ou atypiques.

On va réaliser le modèle complet avec X = body.fat, soit l’IMG de Brozek.

Le modèle s’écrit matriciellement:

\({Y = X \theta } \quad {+} \quad {\epsilon}\)

Ecriture du modèle complet : Modèle M\(_{p+1}\) où \(p=13\)

\[{Y_i = \beta + \sum_{j=1}^{p} + \alpha_jx_{i,j} + \epsilon_i, \quad \epsilon_i \quad iid \quad suit \quad une \quad loi \quad {N(0,\sigma²)}}\]

Sous R :

Model14S = lm(body.fat ~ age + weight + height + neck + chest + abdomen + hip + thigh + knee + ankle + bicep + forearm + wrist)

Les points leviers

Pour une observation i, \(h_{ii}\) est la distance avec le centre de gravité du nuage de points dans l’espace défini par les variables explicatives

Théoriquement, un point levier est un \(h_{ii}\) grand. Cela veut dire que si un point i dépasse un seuil prédéfini, celui-ci devient un point levier.

Nous pouvons comparer \(h_{ii}\) à trois seuils :

  • Le seuil d’Huber (1981)

  • Le seuil de Velleman & Welsh (1981)

  • Le seuil d’Hoaglin & Welsch (1978)

Un point levier est une observation qui ne peut pas être prédite comme les autres observations. Le point levier n’est pas nécessairement influent. Si les observations sont des points leviers et présentent des résidus importants, elles sont susceptibles d’être influentes.


Avec le seuil d’Huber, le point 31 est un point levier.

Avec le seuil de Velleman & Welsh, les points 31, 25, 28, 33, 43, 68, 81, 137 sont des points leviers.

Avec le seuil d’Hoaglin & Welsch, les points 5, 31, 25, 28, 33, 43, 68, 81, 137, 171 sont des points leviers.


Nous allons réaliser le calcul de la distance entre l’observation et la moyenne, l’analyse des résidus pour vérifier ces hypothèses.

La distance de Cook

La distance de Cook mesure l’influence de l’observation i sur l’estimation des paramètres \(\theta\). Elle mesure deux caractères : le caractère aberrant et le caractère levier. Il est reccomandé de supprimer les observations avec une forte distance de Cook.

Nous allons utiliser le seuil : \(\frac{4}{n-p-1}\)

Nous avons les points 31, 68, 65, 137, 175, 171, 163 et 200.


Le graphique “Residuals vs Leverage” ci-dessous va permettre d’identifier les observations influentes dans le modèle de régression multiple. Chaque point est une observation. L’axe x représente le levier, c’est-à-dire qu’elle montre l’ampleur de la modification des coefficients du modèle si une observation spécifique était retirée. L’axe y représente les résidus standardisés, soit la différence standardisée entre une valeur prédite et la valeur réelle pour une même observation.


Interprétation : Selon R, les observations #68 et #137 sont atypiques. L’observation #31 sort de la distance de Cook. Elle dépasse la ligne rouge, alors elle est influente.

L’écart de Welsh-Ku, aussi appelé DFFITS

Le DFFITS est calculé en comparant les valeurs ajustées du modèle lorsque l’observation est incluse ou non du modèle. Une grande distance entre les deux valeurs suggère que l’observation a une influcence importante sur les prédictions du modèle.

L’écart de Welsh-Kuh est défini par :

\[ Wk_i = |t_i^*|\sqrt{\frac{h_{ii}}{1-h_{ii}}}\] Le seuil est \(2 \times \sqrt{\frac{p}{n}}\) :

Le DFBETA est calculé en comparant les coefficients de régression lorsque l’observation est incluse et excluse du modèle.

La fonction influence.measures() de R

La fonction influence.measures() calcule les mesures d’influences selon différentes méthodes pour chaque points. En la combinant à la fonction summary(), on obtient les points influents.

La fonction influcence.measures() donne les mêmes indicateurs mais on aura pas les mêmes résultats car ce ne sont pas les mêmes seuils.

  • DFFITS (seuil = 3p(p + 1)/(n − p − 1).)

  • DFBETA (seuil = 1).

  • les distances de Cook,

Contribution globale des variables explicatives sur body.fat

Modèle linéaire constant

Model1=lm(body.fat~1)

Il s’agit d’effecter le test de Fischer de comparaison de modèles. Dans ce cas on a p = 13 et n = 186.



Hypothèses du test :

(H0) : Modèle M1 contre (H1) : Modèle M11



Statistique de test sous H0 :

\[ F_n = \frac{(SCR(M1) - SCR(M11))/p}{SCR(M11)/(n-p-1)} \sim^{H_0} F(p, n-p-1)\]

Règle de décision (quantile) :

D’après la sortie de “summary(Model14B)”, la statistique de test de Fisher équivaut à 39.6

qf(0.95,13,n-p-1)
## [1] 1.777434

Comme le quantile théorique à 95% d’une F(13,172) vaut 1.78 alors on a fn > 1.78. Par conséquent H0 est rejetée.

Conclusion :

  • On peut plutôt penser avec un degré de confiance de 95% (ou avec un risque d’erreur de première espèce de 5%) qu’il y a un effet significatif de la contribution globale des variables à l’explication de Y.

  • Au moins une des variables contribue à expliquer Y.

  • On préfèrera conserver le modèle complet M14 plutôt que le modèle constant M1. Il n’est pas raisonnable de modéliser ces données par un modèle de régression linéaire.
